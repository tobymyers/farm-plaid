Field and Coordinates API Documentation

Get Fields and Coordinates by User ID

Endpoint: /api/get_fields_coordinates/{user_id}

Method: GET

Returns fields and coordinates associated with the specified user ID.

If the user ID is not found, it returns an error message.

Non-Technical Description

In case you aren't familiar with APIs, you can try it out right now in your browswer by going to https://field-view-a16f43470b9f.herokuapp.com/api/get_fields_coordinates/1 which will return the fields associated with the first user to create a field. Feel free to change the user_id to 2, 3 etc and you will get different data

Here's What Happens

1) Domain Resolution: The browser checks the Domain Name System (DNS) to resolve the domain name (if any) to an IP address. This is like Chrome using its internal phone book to find FieldView's number.

2) HTTP Request: The browser sends an HTTP request to the server identified by the resolved IP address, including the specific path and any query parameters specified in the URL. This is like Chrome making a call to FieldView, asking for the data you requested, in this case information about fields associated with specific users.

3) Server Processing: The server (in this case Heroku) receives the HTTP request, and the application layer (in this case the Flask python app), retrieves the data you are looking for from the PostgresSQL database. The app has received the call from Chrome, and is looking around for the info requested.

4) HTTP Response: The Flask app generates an HTTP response, in this case in the form of JSON, and send the response back to the user's browser. FieldView found what Chrome was looking for and sent it over in a universally understandable format.

5) Client Rendering: The browser receives the response and renders it accordingly. In this case, it gets displayed to you as text and numbers in a Chrome tab, but often another application will ingest the data directly and process it in some other way.
